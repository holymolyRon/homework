# JS 변수, 함수 만들기

1. 변수명과 함수명의 경우 camelCase 표기법을, 상수 변수는 대문자(UPPER_SNAKE_CASE)를 네이밍 컨벤션으로 정해서 일관성을 유지했다.

2. 변수, 상수 작성 과제는 단순히 변수와 상수를 선언하는 연습을 진행했다.

3. 함수 작성 과제의 경우 문제에서 요구하는 사항들을 이해하고, 코드를 작성했다. 약 3시간이 소요되었고, 생각보다 많은 시간이 소요되어서 놀랐다. 요구사항을 이해하고 어떻게 짜야할지 처음부터 고민하면서 작성했고, 테스트를 하는데 시간이 소요되었다.

4. 함수 작성 과제에서 문제에서 매개변수의 값으로 몇가지 경우만 들어갈 수 있을 경우 상수를 객체화했다. 그 이유는 가독성, 오타 방지, 확장성 고려로 인해 상수를 객체화하고 사용했다. 예를들어 회원등급이 VIP와 일반회원 두가지 밖에 없을 경우, `MEMBER_GRADE.VIP`, `MEMBER_GRADE.GENERAL` 와 같이 string을 미리 객체 상수의 프로퍼티의 값으로 선언하여 사용하게 했다. 물론 해당 실습에서는 상수를 객체화 해서 사용하는 횟수가 적어 비교적 효율적이지 않다고 생각했다. 그럼에도 프로젝트가 크다는 가정과 연습의 의미로 상수를 객체화 하고 사용했다. 이것을 고려했기에 시간이 과제 수행 시간이 더욱 오래 걸렸다.

5. 과제를 수행하며 느낀점 : 문제의 요구사항을 차근차근 작성하면서 코드 작성의 감각을 더욱 높이는 연습이어서 좋았다. 코드를 작성하는데 서툴고 느리다는 것을 체감했고, 앞으로 더욱 코드를 직접 치면서 완성하는 연습이 필요하다고 생각했다.
